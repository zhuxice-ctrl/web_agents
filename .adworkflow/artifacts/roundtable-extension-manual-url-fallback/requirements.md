# Extension 模式手动会话网址兜底

## 问题

旧插件桥已连接时，模型标签自动发现仍可能因页面结构、多个会话标签或 adapter 初始化失败而无法绑定。当前 extension 模式只显示“打开/聚焦”和“检测”，没有让用户指定已登录会话标签的入口。

## 预期行为

- 自动发现保持默认路径。
- 每个未绑定模型卡片提供折叠的“手动绑定会话网址”入口。
- 用户粘贴当前 Chrome 中已经打开、已登录的对应模型聊天网址后，可执行精确验证绑定。
- 成功后只保存后端返回的去查询参数、去 hash 网址，不自动导航或自动重新绑定。
- 失败时保留输入并显示登录、人机验证、输入框、adapter 或标签页不匹配原因。

## 非目标

- 不抓取 cookie、token、localStorage、session 响应或账号身份。
- 不自动登录、不处理 CAPTCHA、不创建或导航模型页面。
- 不操作 ChatGPT 真实会话作为本次验收对象。

## 验收标准

1. extension 模式的未绑定卡片可展开 URL 输入和“验证绑定”按钮。
2. URL 必须匹配原 Chrome 中已打开的同供应商标签页，且通过登录、人工验证和输入框检查。
3. 成功绑定使用精确 tabId；查询参数、hash 和认证数据不进入返回值或持久化状态。
4. 自动发现、解除绑定、CDP 手动绑定和现有恢复流程不回归。
5. DeepSeek 与豆包实页至少完成 URL 绑定验证；不发送提示词前再次取得用户确认。
