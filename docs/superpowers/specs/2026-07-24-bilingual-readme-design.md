# 三分支双语 README 设计

## 目标

为 `main`、`webagent`、`tablellm` 三个正式分支建立默认中文、可切换完整英文的仓库入口文档。每个分支使用两个独立文件：

- `README.md`：默认简体中文。
- `README.en.md`：完整英文镜像。

两个文件必须在首屏提供相互切换的相对链接，不依赖脚本、GitHub Pages 或浏览器语言检测。

## 统一导航

中文页首使用：

```markdown
**简体中文** | [English](README.en.md)
```

英文页首使用 `[简体中文](README.md) | **English**`。所有链接使用仓库相对路径，确保分支页面、克隆后的本地 Markdown 预览和 GitHub 均可使用。

## 内容策略

采用中英文完整镜像，而不是英文摘要。两个版本的章节顺序、命令、端口、版本号、依赖关系和安全说明保持一致。允许为语言自然度调整句式，但不得改变产品事实。

所有命令必须来自对应分支的 `package.json` 或已存在的启动脚本。README 不承诺尚未实现的网页适配、权限模式或自动化能力。

## 维护规则

- 改动命令、版本、端口或权限语义时，必须在同一提交中同步两种语言。
- 中文 README 是默认入口，但不是唯一事实来源；两种语言必须同等完整。
- 产品分支不复制另一个产品的详细操作手册，只链接对应正式分支。
- 版本历史使用标签表达，不恢复永久 `*-v1` 分支。
- 不在 README 中记录本机绝对路径、账号信息、真实工作区内容或运行令牌。

## 验证

每个分支完成后执行：

1. 检查 `README.md` 与 `README.en.md` 相互链接存在。
2. 检查两种语言的标题层级和主要章节一一对应。
3. 核对 README 中出现的 npm 脚本均存在于对应 `package.json`。
4. 核对端口、版本和 Core 标签与代码及锁文件一致。
5. 运行对应分支的产品边界检查；`main` 运行 Core 全部测试。
6. 使用 UTF-8 读取文件，确认中英文无乱码且无占位符、`TODO` 或失效的本地链接。
