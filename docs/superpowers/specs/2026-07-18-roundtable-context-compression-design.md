# 圆桌长会话上下文压缩设计

状态：已确认，范围限定为圆桌 MVP。  
日期：2026-07-18

## 目标

在不改写本地原始事件的前提下，控制每次发送给网页模型的圆桌上下文。上下文窗口按 `131072 tokens` 预算计算；估算达到 `80%` 时自动生成结构化压缩状态，目标压缩到约 `20%`。

## 边界

- 只修改 `products/roundtable/**`；插件产品和共享核心不参与本功能。
- `ledger.jsonl` 是唯一权威原始记录，只追加，不覆盖、不删除。
- 压缩状态是可重建的派生投影；每次修订保留版本和审计记录。
- MVP 使用本地确定性规则，不调用东家或其他模型生成摘要。

## 数据模型

在会话 `context.compression` 中保存预算、估算器版本、当前活跃压缩段和历史修订。压缩段记录起止事件索引、事件 ID、`consensus`、`disagreements`、`evidence`、`decisions`、未分类索引、来源事件 ID、创建时间和修订号。

规则只接受明确结构：已有结构化上下文，以及内容中带有明确“共识/分歧/证据/决定”标记的事件。普通观点不提升为共识；无法可靠归类的内容只保留可追溯的短索引。所有摘要项都必须能回指原始事件。

## 触发与投影

发送前根据实际 `buildPrompt()` 组成估算 token，计入固定协议、任务、公共状态、压缩段、近期原文和当前指令。估算值达到 `131072 × 0.8` 时触发压缩，压缩目标为 `131072 × 0.2`。近期原文优先保留约 `16K tokens`；空间不足时按“结构化状态 > 来源引用 > 未分类索引”顺序裁剪，原始记录不受影响。

压缩成功后本次发送直接使用新投影；新事件仍追加到账本，后续只处理压缩边界之后的旧事件。压缩失败时不替换活跃投影，记录失败审计并回退到原始投影。

## 用户修订

页面新增“上下文”视图，区分结构化压缩摘要、压缩覆盖范围和近期原文。用户可以编辑四类结构化摘要及未分类索引；保存会创建新修订，不会修改原始事件。接口提供读取当前压缩状态和提交修订，服务端校验字段类型、来源事件 ID 和会话归属。

## 验收

1. 估算值跨越 80% 边界时只触发一次，目标投影不超过 20% 预算。
2. 压缩前后 `ledger.jsonl` 内容完全一致，压缩版本和修订可恢复、可审计。
3. 规则不会把未标记普通观点伪装为共识、决定或证据。
4. 修订会影响下一次 prompt，旧版本仍可查询。
5. 运行圆桌聚焦测试、`npm.cmd run check:boundaries`、`npm.cmd run test:roundtable` 和 `node --test tools/product-runtime-isolation.test.mjs`。
