# PRD: Web Agents 浏览器插件增强

## 1. 概述

Web Agents 浏览器插件是 `web_agents` 的第一阶段产品入口。它让用户在 ChatGPT、Gemini、DeepSeek、Kimi、Qwen、GLM 等网页 AI 中，以更低门槛连接本地 MCP 工具、管理本地文件权限、插入任务提示词，并在用户显式选择时进行多模型任务分发。

当前仓库已有 `extensions/mcp-superassistant-local-fixed` 作为可用参考版本，但它主要是打包产物，长期不适合作为 UI、双语言和权限系统的主线。因此新版本插件应新建可维护源码工程，当前本地版 MCP SuperAssistant 保留为短期可用版本和行为参考。

## 2. 产品目标

- 默认中文，支持 English 切换。
- 插件首屏以任务入口为主，帮助用户快速把任务插入当前网页 AI 的原生输入框。
- 默认只作用于当前页面，不自动打开全部模型页面。
- 支持用户手动勾选模型并打开子页面，进入多模型任务分发模式。
- 支持 MCP 连接状态、工具列表、权限模式、允许路径和风险提示的可视化管理。
- 支持标准权限模式：路径外可浏览/读取，路径外变更操作必须确认。
- 为后续 GUI 应用、Agent Studio、多模型编排和本地 Agent 框架接入保留接口。

## 3. 用户画像

- 普通用户：希望在网页 AI 里快速使用本地文件能力，不想理解复杂 MCP 配置。
- 进阶用户：希望同时使用多个网页模型，比较回答、整理结论。
- 开发者：希望扩展站点适配器、MCP 工具、权限策略和 Agent 工作流。

## 4. 用户故事

### US-001: 当前页面任务插入

**描述：** 作为用户，我希望在插件里写好任务，并插入当前网页 AI 的原生输入框，以便保留网页原有发送、账号、模型选择和上下文能力。

**验收标准：**

- [ ] 插件默认识别当前网页 AI 站点。
- [ ] 插件显示当前页面是否支持文本插入。
- [ ] 用户点击“插入当前页面”后，任务内容进入网页原生输入框。
- [ ] 默认不自动发送，用户手动确认发送。
- [ ] 插入失败时显示具体错误和建议操作。

### US-002: MCP 连接状态

**描述：** 作为用户，我希望清楚看到本地 MCP 是否已连接、连接地址是什么、当前有多少工具可用。

**验收标准：**

- [ ] 显示连接状态：未连接、连接中、已连接、错误。
- [ ] 显示 Server URI，例如 `http://127.0.0.1:3006/sse`。
- [ ] 显示工具数量，并支持刷新工具列表。
- [ ] 连接失败时显示中文错误解释。
- [ ] 支持 SSE 连接类型，后续可扩展 WebSocket 和 Streamable HTTP。

### US-003: 权限模式管理

**描述：** 作为用户，我希望知道插件和本地工具可以访问哪些路径，并能选择权限模式。

**验收标准：**

- [ ] 默认权限模式为“标准模式”。
- [ ] 允许路径内读写直接执行。
- [ ] 路径外浏览/读取允许执行。
- [ ] 路径外写入、删除、覆盖、移动、重命名、新建目录必须弹出确认。
- [ ] 最高权限模式只能用户手动开启。
- [ ] 最高权限模式开启后，插件持续显示高风险提示。
- [ ] 权限配置从本地配置/本地网关读取，不只保存在 UI。

### US-004: 双语言体验

**描述：** 作为中文用户，我希望插件默认中文；英文用户可以切换 English。

**验收标准：**

- [ ] 默认语言为中文。
- [ ] 支持中文和英文切换。
- [ ] 插件名、按钮、状态、错误、权限提示、工具说明入口都走 i18n。
- [ ] 语言选择持久化保存。
- [ ] 浏览器语言为中文时默认中文；其他语言可默认英文，但仓库发行版默认中文。

### US-005: 多模型显式分发

**描述：** 作为用户，我希望只在需要时手动勾选其他模型页面，并把同一任务分发给多个网页模型。

**验收标准：**

- [ ] 默认只选择当前页面。
- [ ] 插件面板列出可选模型：ChatGPT、Gemini、DeepSeek、Kimi、Qwen、GLM 等。
- [ ] 未打开的模型显示“未打开”，用户点击后才打开子页面。
- [ ] 插件不默认打开所有模型页面。
- [ ] 插入到子页面后，默认仍由用户手动发送。
- [ ] 自动发送只能作为用户主动开启的高级选项。

### US-006: 多模型任务看板

**描述：** 作为用户，我希望多模型并发时能看到每个模型的任务状态，而不需要在多个标签页之间反复寻找。

**验收标准：**

- [ ] 插件显示任务标题和总提示词。
- [ ] 每个参与模型显示一张状态卡片。
- [ ] 卡片状态包括：未打开、已打开、已就绪、已插入、等待用户发送、等待回复、已捕获、出错。
- [ ] 卡片可展开查看发送内容。
- [ ] 捕获到回复后显示摘要和完整回复入口。
- [ ] 完整聊天历史仍以原生网页为准，插件只展示任务级快照和聚合状态。

### US-007: 工具列表和协议能力

**描述：** 作为开发者或进阶用户，我希望看到当前 MCP 服务暴露的 tools/resources/prompts，并能理解哪些能力可用。

**验收标准：**

- [ ] 显示 tools 列表。
- [ ] 显示 tool 名称、描述、输入 schema 和风险级别。
- [ ] 预留 resources 和 prompts 标签页。
- [ ] 对 schema 解析失败、工具为空、连接异常有明确提示。
- [ ] 支持后续扩展到 MCP 更完整协议能力。

## 5. 功能需求

- FR-1: 新建可维护插件源码工程，建议命名为 `extensions/web-agents-extension`。
- FR-2: 插件使用 Chrome Manifest V3。
- FR-3: 插件 UI 使用 React、TypeScript、Vite。
- FR-4: 插件默认语言为中文，并支持 English。
- FR-5: 插件首屏提供任务入口、当前页面状态、MCP 状态和权限摘要。
- FR-6: 插件默认将任务插入当前网页 AI 原生输入框，不自动发送。
- FR-7: 插件支持站点适配器机制，每个网页 AI 站点单独实现输入框检测、插入和页面状态读取。
- FR-8: 插件支持 MCP 服务器配置读取与更新。
- FR-9: 插件支持工具列表刷新、展示和风险分级。
- FR-10: 插件支持标准权限模式作为默认模式。
- FR-11: 插件 UI 修改权限配置时，必须同步到本地配置或本地网关。
- FR-12: 路径外变更操作必须通过本地网关确认流程，不允许只在前端模拟确认。
- FR-13: 多模型分发必须由用户显式勾选参与模型。
- FR-14: 未打开模型页面必须由用户手动触发打开。
- FR-15: 多模型看板展示任务级状态，不替代网页完整聊天界面。
- FR-16: 插件保留高级选项：自动发送、最高权限模式、多模型并发。

## 6. 非目标

- 第一版不做完整桌面 GUI Studio。
- 第一版不默认打开所有模型网页。
- 第一版不强制替代网页原生输入框。
- 第一版不承诺自动抓取所有网页完整历史。
- 第一版不把权限安全只放在浏览器 UI。
- 第一版不强制接入 Claude Code、Codex、Kimi 等 Agent 框架，但保留架构接口。

## 7. 设计考虑

插件首屏建议采用任务工作台布局：

```text
任务入口
  - 输入任务
  - 插入当前页面
  - 多模型分发

当前页面
  - 站点名称
  - 适配状态
  - 输入框状态

本地 MCP
  - 连接状态
  - Server URI
  - 工具数量

权限摘要
  - 当前模式：标准模式
  - 允许路径
  - 最高权限开关状态
```

多模型模式建议采用任务看板：

```text
任务标题
总提示词
参与模型卡片
  - 状态
  - 已插入内容
  - 最新回复摘要
  - 打开原页面
```

## 8. 技术考虑

- 当前打包扩展只作为参考，不继续在大 bundle 上长期迭代。
- 权限判断需要本地网关配合，不能只靠插件 UI。
- 本地网关需要区分只读工具和变更工具。
- 插件侧需要统一站点适配器接口。
- 多模型窗口管理需要记录 tabId、provider、url、adapter 状态。
- 回复捕获应优先做“最新回复快照”，不强行做完整聊天同步。

## 9. 成功指标

- 用户能在 3 分钟内加载插件、连接本地 MCP、向当前网页插入任务。
- 中文用户无需阅读英文说明也能完成基础使用。
- 用户能清楚知道当前权限模式和风险。
- 多模型模式不会在未授权情况下自动打开多个网页账号。
- 新插件源码结构允许后续新增一个网页 AI 适配器而不改核心 UI。

## 10. 开放问题

- 是否优先支持 Chrome Side Panel，还是先用浮动面板加 popup？
- 本地权限网关用 Node.js、PowerShell 包装，还是后续用 Tauri 后端？
- 路径外读取是否需要在隐私模式中额外确认？
- 自动发送是否进入第一版高级选项，还是第二阶段再做？
