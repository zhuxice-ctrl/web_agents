# 仓库开发笔记

这是 `F:\web_agents` 仓库的 Obsidian 开发记录库，服务于持续的代码解析、变更追踪和问题沉淀。

## 记录约定

- 笔记文件名、标题、目录名和 Obsidian 双链统一使用中文；代码中的原始英文名称只在引用源码路径、包名、命令或 API 时保留。
- 按语义功能区分笔记，不按单个文件机械罗列。优先记录模块职责、输入输出、依赖关系、状态变化和边界条件。
- 每次开发工作先写入 `[[开发日志/2026-07-20 首次整理]]` 这类日期日志，再同步更新受影响的模块笔记。
- 模块笔记应链接到上层架构、相关测试、运行命令和风险；发现未知行为时标注“待确认”，不要用猜测补全。
- 代码引用使用仓库相对路径加行号（例如 `products/roundtable/app/server.mjs:1`），以便回到源码核对。
- 链接目标使用中文笔记名，例如 `[[架构总览]]`、`[[模块解析/圆桌工作台]]`；不创建英文别名作为主链接。
- 开发日志记录事实、决策、验证结果和后续动作；不把临时猜测写成结论。
