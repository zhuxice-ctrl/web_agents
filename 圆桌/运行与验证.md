# 运行与验证

## 常用命令

| 目的 | 命令 |
| --- | --- |
| 检查产品边界 | `npm run check:boundaries` |
| 启动圆桌 | `npm run start:roundtable` |
| 圆桌核心测试 | `npm run test:roundtable-web` |
| 圆桌浏览器测试 | `npm run test:roundtable-browser` |
| 圆桌完整测试（含启动器） | `npm run test:roundtable` |
| 插件测试 | `npm run test:plugin` |
| 核心库测试 | `npm run test:core` |
| 全部测试 | `npm run test:all` |
| 本地运行时组合测试 | `npm run test:local-runtime` |

## 端口与进程

- 圆桌工作台：`127.0.0.1:3020`。
- 专用 Chrome CDP：`127.0.0.1:9223`。
- Playwright MCP：`127.0.0.1:8931`。
- 插件文件系统服务：`3006`。
- 插件网关：`3017`。

圆桌默认只拥有 `3020`、`9223` 和 `8931` 生命周期，不依赖插件端口。扩展模式固定使用桥接端口 `3020`，自定义浏览器端口应选择 CDP 模式。

## 首次整理时观察到的测试分布

- `packages/local-core/test` 覆盖原子文件、路径锁、真实路径、权限、事务、工具注册和产品依赖边界。
- `products/roundtable/app` 测试覆盖服务、存储、编排、自动化、MCP 和公共界面模型；另有浏览器端到端测试。
- `products/roundtable/compat-extension/test` 验证扩展协议、页面桥接、内容桥接和后台行为。
- `products/plugin/tests` 与 `products/plugin/services` 覆盖扩展发布边界、权限、文件服务和配置网关。

## 验证记录规则

每次日志注明执行的命令、结果和未执行的验证。浏览器测试需要专用 Chrome 或 fake provider 环境，不能仅凭单元测试结果推断真实页面可用。
