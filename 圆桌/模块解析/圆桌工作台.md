# 圆桌工作台

代码位置：`products/roundtable`，包名 `@web-agents/roundtable-product`。

## 语义职责

圆桌是一个本地多模型协作工作台：把多个模型页面组织为席位，通过回合计划、上下文投影和浏览器自动化完成讨论；同时提供工作区选择、会话持久化、工具调用、产物写入和恢复控制。

## 子模块

### 服务入口与工作区

- `app/server.mjs`：HTTP 服务入口、API 路由、CORS/loopback 校验、运行时组装和中断恢复。
- `app/storage/workspace-registry.mjs`：验证工作区目录、维护最近工作区和当前工作区。
- `app/storage/local-workspace-store.mjs`：在 `.web-agents` 下保存会话、事件、计划、线程、产物和索引。
- `app/core/providers.mjs`：定义 ChatGPT、DeepSeek、豆包等 provider，处理别名、健康状态、设置和席位布局。

### 编排与上下文

- `app/orchestrator/command-parser.mjs`：解析结构化/旧格式指令、`@提及`、模型引用和轮数。
- `app/orchestrator/scheduler.mjs`：创建 turn plan，按回合执行席位，处理暂停、取消、失败恢复和自动收束。
- `context-builder.mjs`、`context-projector.mjs`、`context-compressor.mjs`：构造提示词、按席位投影事件、估算容量并维护压缩摘要。
- `handoff-manager.mjs`：在上下文接近容量时生成交接包并切换线程。
- `reply-contract.mjs`、`quality-analyzer.mjs`：校验回复结构并区分技术失败与质量问题。
- `event-bus.mjs`、`run-registry.mjs`：广播运行事件，跟踪运行中的计划与 AbortController。

### 浏览器自动化

- `app/automation/browser-manager.mjs`：管理 CDP/Playwright 浏览器、页面绑定、导航和 loopback 端点。
- `adapters/*.mjs`：以 provider adapter 封装不同模型页面的选择器、输入和提交行为。
- `worker.mjs`、`controller-tool-worker.mjs`：执行网页回合或受控工具调用，并报告进度。
- `completion-detector.mjs`、`progress-reporter.mjs`：识别新回复是否完成，节流进度事件。
- `extension-relay.mjs`、`extension-browser-manager.mjs`、`extension-worker.mjs`：兼容扩展桥接模式，和默认 CDP 路径隔离。

### 工具循环与产物

- `app/mcp/tool-call-parser.mjs`：从模型文本提取工具调用并格式化工具结果。
- `app/mcp/tool-loop.mjs`：在权限、执行、结果回填之间循环，限制调用次数并处理错误。
- `app/orchestrator/artifact-writer.mjs`：将模型产物写入工作区，外部写入需经过事务管理器。

### 网页界面

- `app/public/app.js`：状态刷新、会话/席位/运行控制、权限与恢复对话框、事件流订阅。
- `conversation-renderer.mjs`：将结构化回复转为安全 Markdown，并限制链接行为。
- `composer-model.mjs`：输入建议、提及 token、目标解析和发送预览。
- `roundtable-physics.mjs`、`workspace-split-controller.mjs`、`detail-sidebar-controller.mjs`：席位布局动画和界面尺寸偏好。

## 默认运行路径

1. `npm run start:roundtable` 启动 `app/server.mjs`，默认监听 `127.0.0.1:3020`。
2. 服务创建 `LocalWorkspaceStore`、浏览器管理器、provider adapters、调度器、权限 broker 和事务 manager。
3. 用户创建会话并选择席位，服务为每个席位建立线程状态。
4. 指令被解析为计划，scheduler 逐回合调用浏览器 worker；结果进入事件总线和持久化存储。

## 当前边界与风险

- 默认依赖专用 Chrome CDP；登录、人机验证和页面选择器失败属于可恢复的运行时状态，不应被当作代码异常吞掉。
- 服务只接受 loopback 请求，并对 API POST 强制 `application/json`。
- 会话恢复会扫描重启前的活动计划；恢复逻辑与持久化状态必须保持幂等。
- UI 与服务共享大量状态字段，新增字段时要同步更新服务序列化、前端渲染和对应测试。
