# 插件与本地网关

代码位置：`products/plugin`。

## 语义职责

本模块包含两部分：可直接加载的 Chrome 扩展，以及独立运行的本地文件系统/权限网关。它服务于旧版或兼容浏览器流程，不属于圆桌工作台的启动生命周期。

## 浏览器扩展

- `extension/manifest.json`：声明扩展权限、脚本和资源。
- `extension/background.js`：后台消息协调和页面事件处理。
- `extension/content/web-agent-result-enhancer.js`：增强网页中的模型结果展示。
- `extension/content/web-agent-insert-fallback.js`：在标准插入路径不可用时提供回退。
- `extension/json_function_call_extractor.js`、`codemirror-accessor.js`、`dragDropListener.js`：解析模型函数调用、接入编辑器和拖放输入。
- `extension/_locales/*`：多语言资源，当前包含中文、英文和韩文。

## 本地服务

- `services/filesystem-stdio-server.mjs`：通过 stdio 暴露文件工具，适合 MCP 进程调用。
- `services/filesystem-http-server.mjs`：提供 HTTP RPC 文件工具入口。
- `services/plugin-gateway.mjs`：插件侧网关，转发工具请求、附加结果和权限上下文。
- `services/config-gateway.mjs`：读取允许根目录，按工具和模式评估权限并生成网关配置。
- `services/permission-store-adapter.mjs`：将插件请求适配到核心库的权限存储。
- `services/start-plugin-services.mjs`：按插件生命周期启动文件服务和网关。

## 端口与边界

- 文件系统 MCP 默认使用 `3006`。
- 插件网关默认使用 `3017`。
- `start-plugin.bat` 只启动插件服务；圆桌的 `start-roundtable.bat` 不应隐式启动它们。
- 两个产品共享 `@web-agents/local-core` 的路径、权限、工具和事务语义，但各自维护配置与运行数据。

## 验证重点

- 扩展发布边界：不应把本地测试数据、服务端代码或非扩展资源打包进发布目录。
- 权限链路：配置网关的允许根目录判断必须和核心库真实路径策略一致。
- 传输链路：stdio、HTTP 和插件网关对工具参数、错误码和结果格式保持兼容。
