# 本地核心库

代码位置：`packages/local-core`，包名 `@web-agents/local-core`。

## 语义职责

本模块为各产品提供与产品无关的本地能力，重点是“能访问什么、谁被授权、如何原子写入、如何回滚”。圆桌和插件都依赖它，核心库本身不依赖产品目录。

## 子模块

### 路径与身份

- `src/path-lock-manager.mjs`：规范化 Windows 路径，生成精确路径、子树路径和源/目标锁键；`PathLockManager` 防止并发操作冲突。
- `src/real-path-policy.mjs`：解析真实路径身份，处理符号链接、重解析点和工作区根目录边界，阻止通过别名绕过限制。
- `src/atomic-file.mjs`：通过临时文件和替换完成原子文件、JSON 与 JSONL 写入。

### 文件工具

- `src/filesystem-tools.mjs`：定义文件读写、目录、移动、删除、搜索等工具；先解析允许目录，再检查目标路径和权限标记。
- `CONTROLLER_TOOL_CAPABILITY` 是受保护的控制器能力，避免任意调用方绕过圆桌控制器直接执行高风险工具。

### 权限

- `src/permission-store.mjs`：落盘保存权限请求、批准/拒绝结果和一次性消费令牌，并用参数哈希绑定请求内容。
- `src/permission-broker.mjs`：面向任务循环的权限协调器，处理请求去重、范围（一次/任务）和审计。

### 工具注册与事务

- `src/tool-registry.mjs`：集中保存工具元数据、参数契约、风险级别和权限要求，提供统一校验。
- `src/transaction-manager.mjs`：记录变更前后状态、路径锁和事务步骤，支持提交、失败恢复、回滚及目标已变化检测。

## 依赖方向

```text
圆桌工作台 ─┐
            ├─> 本地核心库 ─> Node.js 文件系统
插件网关 ───┘
```

核心库不应反向引用 `products/roundtable` 或 `products/plugin`。边界由 `npm run check:boundaries` 与 `packages/local-core/test/no-product-dependencies.test.mjs` 约束。

## 解析重点

- 任何外部写入都应能追溯到权限请求、工具调用和事务记录。
- 路径检查必须同时考虑规范化路径和真实路径，尤其是 Windows 重解析点。
- 事务回滚不是简单覆盖文件；需要比较目标当前状态，避免覆盖用户在事务后做出的修改。
